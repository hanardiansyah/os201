=====================================
a. Persiapan UAS
Kendala :
- Kesulitan dalam memahami beberapa perintah yang diberikan.
- Mahasiswa lupa cara membuat key dan enkripsi.
- Tidak tahu cara melakukan beberapa perintah.
- Server badak mengalami down pada jam tertentu.

Kelebihan :
- Cukup mudah dilakukan.
- Terdapat langkah-langkah yang mudah dipahami.
- Bobot nilai yang diberikan sangat membantu.
- Mudah diakses karena penjelasan berada di Scele CS.
- Waktu yang diberikan cukup lama.
- Terdapat log untuk memantau kesalahan.
- Mempermudah mahasiswa untuk melakukan tahap selanjutnya.
- Mahasiswa menjadi lebih paham terkait dengan pembuatan script.

Kekurangan :
- Kurangnya penjelasan mengenai tujuan persiapan dan gambaran pelaksanaan UAS.
- Langkah pengerjaan ditulis bergabung dengan penjelasan persiapan sehingga terlihat terlalu padat.
- Mahasiswa pada tahap ini cenderung melakukan copy paste langsung dari badak ke github dan tidak melakukan push 
sehingga dapat menyebabkan error.

b. Tahap 1
Kendala :
- Kemampuan penggunaan terminal pada linux yang kurang menghambat proses pengerjaan.
- Kemampuan untuk pembuatan script yang menurut saya sudah cukup advanced yang sangat dibutuhkan disini cukup menghambat proses pengerjaan.
- Terjadi race condition pada saat absen.

Kelebihan :
- Adanya dokumentasi yang diberikan.
- Adanya contoh mengenai github pages yang telah memenuhi syarat. Yaitu, pada github cbkadal.
- Menjadi tahu bagaimana caranya scripting tingkat lanjut, namun bagusnya lagi hal ini tidak menjadi urgensi mahasiswa karena 
mahasiswa tetap dapat menyelesaikan tahap ini dengan melakukan pengecekan secara manual.
- Menjadi tahu bagaimana membuat endpoint baru untuk url pada github pages.

Kekurangan :
- Kurang penjelasan terkait dengan soal yang dimaksud, beberapa seperti contoh SEQNO dan SUMNO yang tidak terdapat pada soal 
Hal ini baru dapat diketahui setelah terdapat mahasiswa yang menanyakan melalui forum di scele.
- Deskripsi tugas seringkali berubah atau revisi selama proses ujian berlangsung.

c. Tahap 2
Kendala :
- Perubahan yang cukup besar antara tahap 1 dengan tahap 2 yang ternyata tahap 2 jauh lebih rumit dibandingkan tahap 1, membuat mahasiswa kebingungan.
- Pada awal pengerjaan sangat terhambat pada implementasi mmap untuk multithreading yang tidak ada pada weekly demo.
- Belum terbiasa membuat program dengan bahasa C.
- Terjadi race condition saat pengumpulan berkas dan absen.

Kelebihan :
- Menjadi lebih tahu bagaimana semafor bekerja untuk menghindari terjadinya race condition dalam mengakses memori yang sama bersamaan.
- Menjadi lebih tahu bagaimana penggunaan semafor biasa dengan semafor untuk mutex.
- Menjadi lebih tahu bagaimana cara memetakan sebuah file ke memory di bahasa C dengan menggunakan mmap.
- Tugas dilakukan secara berkelompok sehingga memudahkan mahasiswa untuk diskusi.

Kekurangan :
- Memahami soal lebih susah dibandingkan memahami langsung dengan melihat programnya, penjelasan yang dijelaskan 
di soal seharusnya dapat lebih jelas tanpa melihat ke program.
- Beberapa seperti “TIME[16]”, nilai dari 16 diperoleh darimana kurang jelas, baru menjadi jelas setelah terdapat salah satu 
peserta ujian yang menanyakan melalui forum di Scele.
- Mekanisme pembagian kelompok yang kurang teratur.
- Deskripsi tugas seringkali berubah atau revisi selama proses ujian berlangsung.
